# VMTools Minecraft 模组重构 PRD

## 1. 项目信息

- **Language**: 中文
- **Programming Language**: Java (Minecraft Fabric 1.21.11)
- **Project Name**: vmtools_refactor
- **原始需求复述**: 用户 Venus 需要一个 Minecraft 1.21.11 Fabric 模组，用于快速传送。当前已有初版代码，但需要专业重构。模组名称为 Venus Mc Tools（VMTools），核心功能是路径点管理工具，支持按快捷键打开管理界面、分组管理、路径点 CRUD、传送命令发送、快捷键系统、导入导出等功能。

## 2. 产品定义

### 2.1 产品目标

1. **P0: 核心传送功能稳定可靠** - 确保路径点传送命令能正确发送到原版服务器，支持 `/res tp`、`/home`、`/warp` 等常见命令格式
2. **P1: 用户体验流畅直观** - 提供暗色主题、圆角卡片、彩色图标的美观 UI，操作逻辑清晰，响应迅速
3. **P2: 数据管理灵活便捷** - 支持路径点分组管理、JSON 格式导入导出，方便用户备份和分享配置

### 2.2 用户故事

1. **As a Minecraft 玩家**, I want to 按快捷键 M 打开路径点管理界面 so that 我能快速查看和管理我的传送点
2. **As a 服务器玩家**, I want to 点击传送按钮自动发送命令 so that 我无需手动输入复杂的传送命令
3. **As a 组织型玩家**, I want to 将路径点按分组管理 so that 我能快速找到常用地点（如村庄、矿洞、基地）
4. **As a 多服务器玩家**, I want to 导入导出路径点配置 so that 我能在不同服务器间共享传送点
5. **As a 个性化玩家**, I want to 为每个路径点设置不同颜色 so that 我能通过视觉快速识别不同类型的地点

## 3. 技术规范

### 3.1 Requirements Pool

#### P0 (Must have)
- 路径点 CRUD（创建/读取/更新/删除）
- 分组管理（创建/删除/重命名分组）
- 传送命令发送（使用 `networkHandler.sendCommand()`）
- 快捷键绑定（默认 M 键打开管理界面）
- 数据持久化（JSON 文件存储）
- 暗色主题 UI

#### P1 (Should have)
- 路径点搜索功能
- 路径点颜色标识
- 使用次数统计
- 传送确认提示（可配置）
- 快速传送快捷键（Ctrl+T）

#### P2 (Nice to have)
- JSON 格式导入导出
- UI 缩放比例配置
- 打开动画时长配置
- 路径点复制功能
- 分组展开/折叠状态记忆

### 3.3 现有代码问题分析

#### 1. 架构问题
- **单例模式过度使用**: `VMToolsClient` 使用静态单例，不利于测试和模块化
- **职责不清晰**: `WaypointManager` 同时负责数据管理和传送逻辑
- **UI 与业务逻辑耦合**: `WaypointScreen` 包含大量业务逻辑

#### 2. 代码质量问题
- **魔法数字**: 颜色值、尺寸值直接硬编码，难以维护
- **重复代码**: 绘制工具方法在多个 Screen 类中重复
- **错误处理不足**: 文件操作缺少完善的异常处理
- **TODO 标记**: 多个功能未实现（导入导出、添加分组界面等）

#### 3. 功能缺失
- **添加分组界面**: `openAddGroupScreen()` 方法未实现
- **导入导出功能**: `importWaypoints()` 和 `exportWaypoints()` 方法未实现
- **输入验证**: 保存路径点时缺少完善的输入验证
- **用户体验**: 缺少操作反馈（成功/失败提示）

#### 4. 性能问题
- **频繁保存**: 每次操作都立即保存到文件
- **搜索效率**: 搜索功能使用线性扫描，大数据量时性能差
- **内存使用**: 所有路径点数据常驻内存

### 3.4 重构建议

#### 1. 架构重构
- **引入依赖注入**: 使用 Fabric 的依赖注入框架
- **分离关注点**: 将数据管理、UI 渲染、业务逻辑分离
- **事件驱动**: 使用事件系统解耦组件

#### 2. 代码重构
- **提取常量**: 将魔法数字提取为配置常量
- **工具类封装**: 创建统一的 UI 工具类
- **错误处理**: 添加完善的异常处理和日志记录
- **输入验证**: 实现完善的输入验证逻辑

#### 3. 功能补全
- **实现添加分组界面**: 创建 `EditGroupScreen`
- **完善导入导出**: 实现文件选择对话框
- **添加操作反馈**: 实现 Toast 通知系统
- **优化搜索**: 使用索引或缓存提高搜索效率

#### 4. 性能优化
- **延迟保存**: 使用防抖机制延迟保存
- **数据分页**: 对大量路径点进行分页显示
- **缓存机制**: 缓存频繁访问的数据

## 4. 待确认问题

1. **Minecraft 版本兼容性**: 确认 1.21.11 版本是否存在特殊 API 变化
2. **服务器权限**: 确认原版服务器是否限制 `/res tp` 等命令的使用
3. **数据迁移**: 现有用户数据如何迁移到重构后的版本
4. **多语言支持**: 是否需要支持英文等其他语言
5. **模组分发**: 是否需要发布到 CurseForge 或 Modrinth 等平台
6. **测试策略**: 如何测试与原版服务器的兼容性
7. **性能要求**: 大数据量（1000+ 路径点）下的性能要求
